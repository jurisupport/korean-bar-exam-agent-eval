# RUNBOOK_X — 클린룸 조건 F++ 복제군 (GPT-5.6 Sol, xhigh) · 회차별 별도 세션 실행

**이 클린룸(`cleanroom_X_gpt56sol/`)은 자급자족이다.** 다른 세션에서 이 폴더 하나만 보고 그대로 실행한다.
세션마다 이렇게 지시: **"cleanroom_X_gpt56sol 폴더의 RUNBOOK_X.md대로 실행해줘. phase=F0|F1, run=N"**.
- 기존 봉인 클린룸(M·M+·M++·M+++)과 **무관**(비교·재사용 안 함). 그 회차들의 **채점된 교훈만** 설계에 반영했다(아래 근거표).
- 목표 = "확신 있게 틀리지 않기 + 정답을 오답으로 안 덮기 + 천장 정직 라벨링". 40/40은 보장이 아니라 가능성.

## 설계 근거 — 기존 10개 조건·25회차에서 채점으로 확인된 것만 채택

|---|---|

| **F0 오염 점검 선행** | 평가 대상 모델에 시험·가안이 학습 또는 회상됐을 가능성을 별도 측정. OFF 베이스라인과 원형 F의 사전등록 기준을 그대로 유지 |
| 봉인 즉시 **쓰기잠금(chmod 444) + 스냅샷** | 봉인본 사후 교체 사고 2회(A_off ③→⑤, Mplus2 21:08 40/40 → 21:27 38/40) 재발 방지 |

폐기(채택 안 함): 판본 컷오프 규칙(GPr 87.5%, 음성) / 전면 레드팀(M+ 천장 그대로+회귀) / 이종모델 앙상블(v4 반증) / 판례DB 1차 투입(G+P 반증).

## 설계 3원칙 (어기면 무효)
1. **정답표 비튜닝(blind)** — 답을 알고 라우팅/규칙/지문정오/프레이밍을 맞추지 않는다. 고치는 건 "틀리는 메커니즘".
2. **메커니즘 교정** — 상관편향→프레이밍 3표 / 국소화실패→지문단위 / 과교정→블라인드 flip가드 / 증거공백→딥다이브 소스 확장.
3. **재현성 바** — "안정적 40" = 독립 3회(별도 세션) 봉인본 모두 40/40. 1회 40은 분산일 수 있어 불인정.

## 모델 고정 (이 클린룸의 핵심 변수)
- 솔버·딥다이브 전부 **`gpt-5.6-sol` + `model_reasoning_effort="xhigh"`**. 세션 기본모델·기본 effort에 의존하지 말 것.
- 각 슬롯은 대화형 에이전트 스폰이 아니라 아래 고정 옵션의 **독립 비대화 호출**로 실행한다. 15개 슬롯은 각각 새 `codex exec` 프로세스이며 세션을 재개·공유하지 않는다.
  `codex exec -m gpt-5.6-sol -c 'model_reasoning_effort="xhigh"' --ephemeral --json -s workspace-write -C cleanroom_X_gpt56sol -`
- `~/.codex/config.toml`의 `korean-law` MCP 등록을 사용해야 하므로 **`--ignore-user-config`를 붙이지 않는다**. 다른 등록 MCP·도구는 프롬프트의 허용 목록에 따라 사용 금지다.
- **모델 검증**: 각 솔버가 산출 JSON `"model"` 필드에 자기 실제 모델명을 적는다. F1은 투표 전에 15개 전부 `gpt-5.6-sol`인지 `f_vote.py`가 확인하고, F0는 봉인 전에 5개 전부를 `f_merge_seal.py`가 확인한다. 하나라도 다르면 중단하고 그 슬롯만 같은 고정 옵션으로 재실행한다.

## Phase F0 — 오염 점검 + 베이스라인 (F1보다 반드시 먼저)

**목적**: GPT-5.6 Sol(xhigh) 순수지식 점수 측정. 시험·가안 노출 가능성이 있으므로,
OFF가 높으면 "도구 기여" 논증 전체의 프레임이 바뀐다. **판정 기준은 지금 여기서 사전등록한다**:

> **오염 판정(사전등록)**: Foff 3회 평균 ≥ **38/40** → "회상 오염 의심" 플래그.
> 이 경우 F1 결과 보고는 OFF<ON 격차 논증을 **주장하지 않고**, 근거 인용 역추적(authorities→정답 도출 감사) 중심으로 프레임 전환한다.
> 참고 베이스라인: Opus 4.8 OFF 5회 평균 34.6/40(86.5%).

- 조건: **외부도구 전무**(MCP·curl·라이브검색·WebSearch 전부 금지). 순수 모델 지식만.
- 8문항×5배치 × 3회(Foff1·Foff2·Foff3). 배치당 독립 `codex exec` 1개(고정 모델·effort), 프롬프트 `tools/f_off_prompt.md`.
- 회차별 5콜 실행: `bash tools/x_run_slots.sh off {run}`. 이 런처는 풀이를 합치지 않고 슬롯별 비대화 프로세스·JSONL 로그를 각각 남긴다.
- 각 run 봉인: `python3 tools/f_merge_seal.py Foff{run}`. 3회는 별도 세션 권장(최소한 타 회차 산출물 미참조).

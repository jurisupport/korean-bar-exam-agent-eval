# 재현성과 무결성

## 재현 범위

이 배포본은 두 가지 재현 목표를 구분합니다.

1. **결과 재계산**: 포함된 봉인본·중간 투표·딥다이브 패치·정답표로 점수와 단계별 효과를 다시 계산합니다. 이 목표는 저장소만으로 재현 가능합니다.
2. **프로토콜 재실행**: 모델을 처음부터 다시 호출해 새 답안을 만듭니다. 동일 모델·Codex 설정·MCP·로컬 DB·시점 차단을 별도로 구성해야 하며, 확률적 출력의 비트 단위 일치는 보장하지 않습니다.

결과 재계산은 다음 명령으로 수행합니다.

```bash
python3 analysis/analyze_results.py
```

배포 파일 무결성은 macOS에서 `shasum -a 256 -c manifests/SHA256SUMS.txt`, Linux에서 `sha256sum -c manifests/SHA256SUMS.txt`로 확인할 수 있습니다.

## 포함 범위

`experiment/`에는 다음이 포함됩니다.

- 사전등록과 런북
- 고정 프롬프트와 실행·투표·봉인·채점 스크립트
- 문제 픽스처와 최종 정답표
- Foff1–3, F1–3 봉인본과 SEAL
- 프레이밍별 part, 투표·딥다이브 패치
- 회차별 `_runs` 스냅샷
- 최초 블라인드 채점 출력과 원 결과 보고서

원시 `02_logs/*.jsonl`은 Git 본문에서 제외했습니다. 로그에는 로컬 경로와 법령·판례·교재 응답 전문이 포함되고 전체 크기가 100MB 이상이기 때문입니다. 원 로그의 파일별 SHA-256은 `manifests/RAW_LOGS_SHA256.txt`에 있습니다.

## 원 실험 실행 요건

정확한 재실행에는 저장소 파일 외에 다음 환경이 필요합니다.

- Codex CLI와 `gpt-5.6-sol`
- `model_reasoning_effort="xhigh"`
- `korean-law` MCP
- 교재DB 8768: 하이브리드, 임베딩 17,167개
- 판례DB 8770: 하이브리드, 임베딩 34,788개, `EXAM_MAX_DATE=2026-01-05`
- 법고을·최근 판례검색 스킬

실험 당시는 `legal-books` 툴킷에 실험 전용 로컬 DB `books_bar.db`를 별도로 연결했습니다. 이 DB는 JuriSupport 플러그인에 포함된 데이터가 아니며, 18권, 17,167개 청크로 구성해 8768 포트에서 제공했습니다. 원시 실행 로그의 검색 결과에 등장한 공법 서적은 김하열 『헌법강의』 제7판, 이동흡 『헌법소송법』 제4판, 헌법재판소 『헌법재판실무제요』 제3개정판, 하명호 『행정법』·『행정쟁송법』 제8판, 김해마루 『법학 입문(공법 - 헌법 행정법)』 제5판입니다. 세부 서지정보는 README에 있습니다.

이 서지정보는 검색 코퍼스 감사용입니다. 교재 본문·스캔·OCR·`books_bar.db`·임베딩은 배포본과 JuriSupport 플러그인 저장소 모두에 포함되지 않습니다.

배포본의 `experiment/tools/f_serve.sh`는 원 실험본에 하드코딩되어 있던 사용자 로컬 DB 절대경로를 제거했습니다. 재실행 시 `KDB`를 판례DB 디렉터리로 설정해야 합니다.

```bash
KDB=/path/to/precedent-db bash experiment/tools/f_serve.sh
```

## 새 블라인드 재실행

배포본의 `experiment/` 안에는 이미 공식 정답표, 봉인 답안, 채점 결과, 스냅샷이 있습니다. 따라서 배포본 안에서 솔버를 실행하면 새로운 블라인드 실험으로 인정할 수 없습니다.

새 실험은 `00_fixtures/`, `tools/`, `PREREG_X.txt`, `RUNBOOK_X.md`만 넣은 새 클린룸을 만들어 시작해야 합니다. 이 클린룸은 배포 저장소, 정답표, 기존 봉인본, Git 이력을 읽을 수 없는 컨테이너·VM·별도 사용자 환경에 놓아야 합니다.

실행 전에는 다음을 확인합니다.

- `PREREG_X.txt`와 픽스처를 읽기 전용으로 고정
- 빈 `01_sealed/_parts`, `02_logs`, `03_answer_key`, `04_scoring`, `_runs` 디렉터리 생성
- 솔버 가시 범위 밖의 운영자 전용 위치에 정답표 보관
- 8768·8770 서버의 임베딩 수와 8770의 `max_date` 확인
- 전 호출에 `gpt-5.6-sol`, `model_reasoning_effort="xhigh"`, `--ephemeral` 적용

이후 실행 순서는 `experiment/RUNBOOK_X.md`를 따릅니다. Foff1·Foff2·Foff3을 먼저 봉인하고, F1·F2·F3을 서로 별도 세션에서 봉인한 뒤, 여섯 회차가 모두 봉인됐을 때만 정답표를 복구해 채점합니다.

원 실험의 회차별 딥다이브 보조 스크립트 일부는 당시의 경합 문항 목록을 기준으로 작성됐습니다. 새 실험에서는 `f_vote.py`가 생성한 새 경합 목록을 따라야 하므로, 이 부분을 기계적으로 일반화하고 실행 전에 고정해야 합니다. 현재 배포본은 결과 감사에는 충분하지만 임의의 새 출력을 위한 완전한 원클릭 재실행 패키지는 아닙니다.

독점 모델과 외부 서비스가 포함되므로 미래의 비트 단위 재현은 보장할 수 없습니다. 이 저장소가 보장하는 것은 당시 입력·규칙·봉인 산출물·해시·채점 결과에 대한 감사 가능성입니다.

## 포함 산출물의 채점 재실행 금지

원 실험의 `f_grade.py`는 정답표 최초 개봉과 함께 정확히 한 번 실행됐습니다. 이 배포본에 포함된 기존 봉인본에 `f_grade.py`를 다시 실행하며 새 블라인드 실험이라고 주장해서는 안 됩니다. 새 채점은 위의 격리 절차로 생성한 여섯 새 봉인본에 대해, 전 회차 봉인 후 한 번만 수행해야 합니다.
